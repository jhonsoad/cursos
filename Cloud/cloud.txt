Modelos de cloud: 

IaaS - Infrastruture as a Service
A infraestrutura como serviço oferece recursos de computação, armazenamento e rede. Ou seja, esse modelo disponibiliza um ambiente com uma infraestrutura virtualizada. Ex; Amazon Web Service e Google Compute Engine


PaaS - Platform as a Service
Na Plataforma como serviço, temos um programa ou aplicativo entregue com o objetivo de facilitar a implementação de algum serviço,; ou seja, é criado um ambiente para programação e o deesenvolvimento de alguma solução. Ex: Heroku e Red Hat OpenShift
 
SaaS - Software as a Service
No software como serviço, o software é executado em um servidor, de forma que o cliente não precisa tê-lo instalado em sua maquina. Ex: Gupy e Trello

Armazenamento
S3 - repositorio para armazenamentode de arquivos

EBS - discos de armazenamentos para SO

EFS - comartilhamento de arquivos

Classes de armazenamento - disponibilidade dos arqivos ex: a qualquer momento ou em 1 hora 10 horas para backup do backup arquivos que vc não precisa que estejam disponiveis rapido custo menor
 
S3Intelligent-Tiering - economia automatica para dados com acessos desconhecidos ou variaveis

S3 Standard - para dados acessados com frequencia

S3 Standard-Infrequent Access (S3 Standard-IA) e S3 One Zone-Infrequent Access (S3 One Zone-IA) - para dados acessados com menos frequencia

S3 Glacier Instant Retrieval - dados que precisam de acesso imediato

S3 Glacier Flexible Retrieval (anteriormente S3 Glacier) - dados de longo prazo raramente acessados que não requerem acesso imediato

S3 Glacier Deep Archive (S3 Glacier Deep Archive) - armazenamento de longo prazo e preservação digital com recuperação em horas com o menor custo de armazenamento na nuvem

VPC - Virtual Private Cloud
é um servico que permite criarmos uma rede virtual isolada que funciona igual uma rede local de empresa, podemos configurar como queremos
por default se tivermos isntancias na mesma regiao elas teram conectividade mas se estiverem em regiões diferentes é necessário criar um roteamento para que elas se comuniquem
a VPC default que vem para cada região é diferente e o número de subnets é uma para cada zona de disponibilidade que tem na região
NAT Gateway - Network Address Translation
usada para que instancias em subnets privadas possam se comunicar com servicos fora da rede mas os servicos externos não podem iniciar comunicação com essas intancias

IP Elastic
