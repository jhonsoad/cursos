3 funções principais do jest: describe, test e expect
describe é a descrição da switch de teste onde ficam os teste, switch de teste é uma bateria de teste de uma funcionalidade
posso usar test ou it para colocar o meu teste unirtario
expect onde informo o que quero testar e com .toBe o retorno esperedo

describe('Calculate area of square', () => {
    test("passed the value 3, it should return 9", ()=>{
        expect(areaOfSquare(3)).toBe(9)
    })
})teste para uma função que eleva um numero a potencia 2
o .toBe() compara dse um objeto é === o toEqual verifica mais a fundo a igualdade de objetos por exemplo verifica se são iguais os valores de cada uma propriedade dentro de um objeto

test("not passing the paramiter should return null",
()=>{
	expect(factory.getPerson()).toBeNull()
})
toBenull() é usado para verificar se o valor retorna nulo

test("passed a string with the client value, it should return a new client object", ()=> {
	expect(factory.getPerson("Client")).toBeInstanceOf(Client)
})
toBeInstanceOf() é usado para verificar se um objeto ou metodo esta sendo instanciado por uma classe, factory é uma const com o valor de uma classe esta declarado no teste, e chama outra classe que é Client que esta sendo importada no teste 

toBeUndefined() verifica se uma propriedade é undefined
toBeDefined() verifica se uma propriedade foi definida, mesmo que ela exista no objeto ela tem que ser declarada no test

var product = new Product()
product.description = "Biscoito Trakinas"

test("should return that the stock property is undefined", ()=> {
	expect(product.stock).toBeUndefined()
});

test("should return that the description is defined", ()=> {
	expect(product.description).toBeDefined()
}); mesmo criando o product temos que definir a propriedade description
